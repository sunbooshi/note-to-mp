### 数学公式优化
在之前版本中，数学公式默认会缩小一号，现在默认不缩小。

如果需要缩小，可以通过如下自定义CSS进行调整。

````
``` less
.note-to-mp {
	.block-math-svg {
		svg {
			font-size: 0.85em;
		}
	}

	.inline-math-svg {
		svg {
			font-size: 0.85em;
		}
	}
}
```
````

如果文章中公式较多，可以尝试修改段落的行高来调整公式显示效果。

````
``` less
.note-to-mp {
	p {
		line-height: 1.8em;
	}
}
```
````

### 支持 𝕏（推特）文章
> [!WARNING]
> 目前该功能还无法完全支持 𝕏 文章编辑器，在某些情况下存在格式丢失的风险!
> 请谨慎使用！

现在可以将Obsidian笔记发布为 𝕏（推特）长文章。

**使用前提：**
需配合「[文兔兔助手(v0.0.4)](https://chromewebstore.google.com/detail/cefclimhdhgbknphcabhpnmmcnpjlajg)」浏览器插件使用。

**使用方法：**
在插件中复制内容后，在chrome中打开 𝕏 文章编缉页面，通过文兔兔助手的**同步到 𝕏（推特）文章**功能，将笔记同步到 𝕏 文章编辑器中。

**目前支持的内容格式：**

* 标题
* 副标题
* 正文
* 图片
* 引用
* 列表

**暂不支持：**

* 代码块
* 分割线
* 数学公式

> [!WARNING]
> 如果笔记中包含图片，可能出现部分内容格式丢失的情况。
> 
> 另外，图片上传耗时较长，请耐心等待图片上传完成。


### 工作流支持（自动化创作升级）

如果你正在使用 n8n、Coze 等工作流平台，现在可以直接与 NoteToMP 打通，实现：

* AI 自动生成内容
* 发布前 / 发布后自动执行额外动作
* 通过『**NoteToMP:运行工作流**』命令手动触发工作流

#### 安全获取公众号 AccessToken（会员功能）

在工作流中调用公众号 API 时，通常需要暴露 AppSecret，存在安全风险。

现在插件提供专用接口来获取 AccessToken：

* 不需要在工作流中暴露 AppSecret
* 不需要固定服务器 IP
* 更安全、更简单

详细教程：[如何使用AI工作流为公众号创作赋能？](https://mp.weixin.qq.com/s/M2aL0l-NOn15T7YoAO413A)


#### 在笔记中直接触发工作流

你可以在笔记里插入一个 `workflow` 代码块，在发布前或发布后自动调用外部工作流。

示例：

````markdown
```workflow
url: https://api.coze.cn/v1/workflow/stream_run
runAt: start
method: POST
header:
  authorization: Bearer pat_12345
  content-type: application/json
body:
  workflow_id: "123456789"
  enabled: true
  parameters: 
    title: $title
```
````


#### 调用参数说明

* `url`：请求地址
* `runAt`：执行时机
  * `start`：发布前执行
  * `end`：发布后执行
* `method`：GET 或 POST
* `header`：请求头
* `body`：请求体


#### 支持动态引用笔记内容、笔记属性等

工作流参数支持直接引用当前笔记属性或仓库内容：

```yaml
body:
  workflow_id: "123456789"
  enabled: true
  parameters: 
    author: $author
    content: ![[提示词]]
    image: ![[cover.jpg]]
```

说明：

* `$author` → 当前笔记的 author 属性
* `[[提示词]]` → 引用名为“提示词”的笔记内容
* `![[cover.jpg]]` → 引用仓库中的图片文件

通过该功能可以将仓库中的笔记、图片等内容，作为参数传递给工作流，实现更灵活的自动化创作。

### 超宽表格显示优化
新版本中超宽表格会显示滚动条，用户可以滚动查看表格内容。

**⚠️注意：**

由于Markdown中无法设置表格中单元格的宽度，正常情况下表格会被压缩，导致单元格宽度很小，用户体验不佳。

可以通过在表头中增加`&nbsp;`的个数来设置单元格的最小宽度。

示例：

````markdown
| 表头1&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; | 表头2&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; |
| --- | --- |
| 内容1 | 内容2 |
````


### 自定义文章样式
新版本支持使用`less`定义文章样式，并且新增了`自定义`主题，启用`自定义`主题后，将完全使用用户自定义的样式，不再受默认样式影响，用户可以完全控制文章的样式。

详细的使用指南请参考：[自定义文章样式](https://docs.dualhue.cn/customcss)

```
}
.fnref-sup::after {
	content: ']'
}
```
